---
title: 'Speko'
description: 'One API for every voice provider. Speko benchmarks STT, LLM, and TTS in real time and routes each call to the best provider for your language, vertical, and latency/cost target.'
---

Speko is a voice gateway. You ship one integration; we route each request to the highest-scoring provider for your `(language, vertical, optimizeFor)` intent. Failover is server-side. Providers rotate without a code change.

## Three ways to integrate

<CardGroup cols={3}>
  <Card title="One-shot HTTP" icon="terminal" href="/guides/one-shot">
    Call `/v1/transcribe`, `/v1/synthesize`, `/v1/complete` directly. Best for batch jobs and server pipelines.
  </Card>
  <Card title="Real-time voice agent" icon="microphone" href="/guides/voice-agent">
    LiveKit-based agent worker. Browser ↔ agent ↔ Speko. STT/LLM/TTS pipeline runs in your worker.
  </Card>
  <Card title="Browser conversation" icon="browser" href="/guides/realtime-conversation">
    `@spekoai/client` joins a Speko session over WebRTC. Mic in, agent voice out, transcripts on the data channel.
  </Card>
</CardGroup>
