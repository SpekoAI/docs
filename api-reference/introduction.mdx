The Speko REST API is hosted at `https://api.speko.dev`. All endpoints are under `/v1/`. Requests and responses are JSON unless noted (transcribe accepts binary audio; synthesize returns binary audio).

## Base URL

```
https://api.speko.dev
```

## Authentication

Every endpoint requires a bearer API key in the `Authorization` header:

```http
Authorization: Bearer sk_live_...
```

Mint a key in the dashboard under **Settings → API keys**.

The OAuth-based session token issued by `POST /v1/sessions` is **not** an API key — it's scoped to a single LiveKit room and is used by the browser SDK, not for general REST calls.

## Errors

Errors return `application/json`:

```json
{
  "error": "human-readable message",
  "code": "MACHINE_READABLE_CODE"
}
```

Common codes:

| HTTP | Code | Meaning |
| --- | --- | --- |
| `400` | `INVALID_AUDIO`, `INVALID_INTENT`, `VALIDATION_ERROR` | Malformed request. |
| `401` | `UNAUTHORIZED` | Missing or invalid API key. |
| `429` | `RATE_LIMITED` | Backoff, then retry; respect `Retry-After`. |
| `5xx` | `TRANSCRIBE_FAILED`, `SYNTHESIZE_FAILED`, `COMPLETE_FAILED`, `ALL_PROVIDERS_FAILED`, `SESSION_CREATE_FAILED` | Upstream / routing failure. |

## Routing headers

`/v1/transcribe`, `/v1/synthesize`, `/v1/complete` all return:

- `X-Speko-Provider` — provider that handled the call
- `X-Speko-Model` — specific model
- `X-Speko-Failover-Count` — number of providers tried before success
- `X-Speko-Scores-Run-Id` — benchmark snapshot id used for routing
- `/v1/synthesize` additionally: `X-Speko-Audio-Format`

Log these in production. They're how you correlate behavior with routing decisions.
