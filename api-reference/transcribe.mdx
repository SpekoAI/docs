## POST /v1/transcribe

Speech-to-text. Body is binary audio. Routing intent goes in the `x-speko-intent` header.

### Request

```http
POST /v1/transcribe
Authorization: Bearer sk_live_...
Content-Type: audio/wav
x-speko-intent: {"language":"en-US","vertical":"general"}
x-speko-constraints: {"allowedProviders":{"stt":["deepgram"]}}
```

Body: binary audio. Supports WAV, MP3, raw PCM, anything the underlying provider accepts.

| Header | Required | Notes |
| --- | --- | --- |
| `x-speko-intent` | yes | JSON `{ language, vertical, optimizeFor? }`. |
| `x-speko-constraints` | no | JSON `{ allowedProviders: { stt: string[] } }`. |
| `Content-Type` | recommended | Audio MIME type for the underlying provider. |

### Response (`200 OK`)

```json
{
  "text": "...",
  "provider": "deepgram",
  "model": "nova-2",
  "confidence": 0.94,
  "failoverCount": 0,
  "scoresRunId": "..."
}
```

Plus headers: `X-Speko-Provider`, `X-Speko-Model`, `X-Speko-Failover-Count`, `X-Speko-Scores-Run-Id`.

### Errors

| HTTP | Code | When |
| --- | --- | --- |
| `400` | `INVALID_AUDIO` | Empty body, missing/invalid intent header, malformed constraints. |
| `5xx` | `TRANSCRIBE_FAILED` | No STT provider for intent, or every candidate failed. |
