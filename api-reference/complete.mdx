## POST /v1/complete

Single-turn LLM completion. Returns the assistant's reply text + token usage.

### Request

```http
POST /v1/complete
Authorization: Bearer sk_live_...
Content-Type: application/json
```

```json
{
  "messages": [
    { "role": "system", "content": "You are concise." },
    { "role": "user", "content": "Hi!" }
  ],
  "intent": { "language": "en", "vertical": "general", "optimizeFor": "balanced" },
  "temperature": 0.7,
  "maxTokens": 256,
  "systemPrompt": "Override or supplement system role here.",
  "constraints": { "allowedProviders": { "llm": ["openai"] } }
}
```

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `messages` | array | yes | Min 1. Each `{ role: "system" \| "user" \| "assistant", content: string }`. |
| `intent` | object | yes | Standard routing intent. |
| `temperature` | number | no | `0..2`. |
| `maxTokens` | int | no | |
| `systemPrompt` | string | no | Forwarded to provider via the canonical system-prompt slot. Don't *also* prepend the same content as a `system` message; some adapters honor both and double up. |
| `constraints` | object | no | Standard. |

### Response (`200 OK`)

```json
{
  "text": "Hello!",
  "provider": "openai",
  "model": "gpt-4o-mini",
  "usage": { "promptTokens": 14, "completionTokens": 4 },
  "failoverCount": 0,
  "scoresRunId": "..."
}
```

Plus the standard `X-Speko-*` routing headers.

### Errors

| HTTP | Code | When |
| --- | --- | --- |
| `400` | `VALIDATION_ERROR` | Body fails Zod validation. |
| `5xx` | `COMPLETE_FAILED` | No LLM provider for intent, or every candidate failed. |
